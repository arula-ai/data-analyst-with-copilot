**Project Overview: GitHub Copilot for Data Analysis — Hands-On Lab**

**Lab Type:** Instructor-Led Training (ILT) | Blended Audience: Engineers & Analysts
**Duration:** 90 minutes
**Offering:** Weekly, 4x per month
**Focus:** Using GitHub Copilot in VS Code to profile, clean, query, and visualize operational financial data — responsibly and accurately.
**Approach:** Scenario-based, artifact-driven lab using the RIFCC-DA (Responsible, Iterative, Framework-driven Copilot Collaboration for Data Analysis) prompting framework.
**Format:** 3 parallel sub-labs — participants choose one scenario and work through it end-to-end.

---

**Three Scenarios (Sub-Labs):**

1. **Sub-Lab A — Treasury Anomaly Detection** (Meridian Asset Management)
   - Dataset: `data/treasury_payments.xlsx` — 500 Q4 2024 treasury payment records, 14 columns
   - 11 intentional data quality issues (documented in `data/treasury_schema.md`)
   - Goal: Detect abnormal payment patterns, surface anomalies, produce 3 interactive HTML charts
   - PII-adjacent field: `counterparty_masked` — never include in outputs

2. **Sub-Lab B — Root Cause Analysis** (Orion Payments Inc.)
   - Dataset: `data/rca_app_logs.csv` — 300 application log entries, 9 columns
   - Pre-step: Review `data/app_service.py` (5 microservices with intentional BUG comments)
   - 4 data quality issues (documented in `data/rca_schema.md`)
   - Goal: Identify root cause service from log patterns, produce 3 charts
   - PII-adjacent field: `user_id_masked` — never include in outputs

3. **Sub-Lab C — Product Modernization** (Centrix Financial Systems)
   - Dataset: `data/mainframe_usage.xlsx` — 400 feature usage records, 10 columns
   - Pre-step: Review `data/legacy_mainframe.py` (6 legacy functions with intentional BUG comments)
   - 4 data quality issues including sentinel 9999 (documented in `data/mainframe_schema.md`)
   - Goal: Prioritize legacy features for modernization, produce 3 charts
   - No direct PII — Internal classification tier

**Repository Structure:**

- **`data/`** — Source datasets (xlsx, csv), schema files, and pre-step source code
- **`scenarios/`** — 3 sub-lab folders (sub-lab-A-treasury, sub-lab-B-rca, sub-lab-C-modernization)
  - Each contains: `SCENARIO_BRIEF.md`, `SUB_LAB_GUIDE.md`, `exercises/` (flawed analysis + SQL reference + answer key)
- **`.github/agents/`** — 6 custom Copilot agent mode files (data-profiling-analyst, data-cleaning-engineer, exploratory-data-analyst, visualization-architect, data-risk-reviewer, responsible-use-auditor)
- **`.github/prompts/`** — 6 named Copilot prompt files (invoked with `/` slash commands)
- **`outputs/`** — Participant deliverables saved here (profile.md, cleaning_decisions.md, chart_*.html)
- **`scripts/`** — Cleaning scripts generated in Phase 2 (profile_*.py, clean_*.py, visualize_*.py)
- **`templates/`** — Structured templates for each phase deliverable
- **`reference/`** — RIFCC-DA framework, responsible use policy, glossary, Copilot commands, notebooks guide
- **`LAB_ACTION_GUIDE.md`** — Primary participant runbook (all 3 scenarios, all phases)
- **`QUICK_START.md`** — VS Code extension setup + Copilot Chat orientation in under 5 minutes
- **`VERIFY_BEFORE_SEND.md`** — Pre-prompt data privacy preflight checklist
- **`FACILITATOR_GUIDE.md`** — Instructor reference (ground truths, timing, debrief script, known failure patterns)
- **`.devcontainer/`** — GitHub Codespaces configuration (Python 3.11, auto-installs requirements)
- **`GitHub_Copilot_for_Data_Analysis.pptx`** — 90-minute presentation deck

**Lab Workflow (90 minutes total):**

| Block | Time | Content |
|-------|------|---------|
| Shared Demo | 0–30 min | Facilitator-led: workspace setup, RIFCC-DA framework, iterative prompting demo, first agent walkthrough |
| Scenario Selection | 30–32 min | Participants choose Sub-Lab A, B, or C |
| Scenario Sprint | 32–82 min | Phase 1 (10 min: profile) → Phase 2 (25 min: clean + EDA) → Phase 3 (15 min: visualize + export) |
| Group Debrief | 82–90 min | Round-robin: 1 finding + 1 risk caught + 1 Copilot correction per participant |

**Key Principles:**
- Use Copilot Chat agents (selected via dropdown) — not `@` syntax
- Reference files with `#filename` in prompts (schema always attached alongside data)
- Follow RIFCC-DA framework: Role, Inputs, Format, Constraints, Checks
- Validate every generated script before running — no silent data drops allowed
- Sentinel values (999, -1, 9999) must be excluded from all calculations
- PII-adjacent fields (`counterparty_masked`, `user_id_masked`) never in any output, chart, or print
- All deliverables saved to `outputs/` using provided templates

**Technical Requirements:**
- Python 3.10+ with pandas, numpy, openpyxl, plotly
- VS Code with GitHub Copilot Chat, Python, Jupyter
- No external databases — all data is xlsx/csv
- GitHub Codespaces supported via `.devcontainer/`
